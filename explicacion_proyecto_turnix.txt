EXPLICACION DEL PROYECTO TURNIX
===============================

INTRODUCCION
------------

Hola, este proyecto se llama Turnix. Es un sistema de gestion de filas, citas y clientes para pequenos negocios.

El proyecto esta dividido en dos partes principales:

TURNIX-javaAV/
  frontend/
  backend/

La carpeta frontend contiene todo lo que ve el usuario en la pagina web. Esta hecho con React + Vite.

La carpeta backend contiene la API. Esta hecha con Node.js + Express y se conecta a MySQL.


PAGINA PRINCIPAL
----------------

La pagina principal, o index, ya no se modifica desde index.html como antes. Ahora usamos React.

El archivo base del navegador esta aqui:

frontend/index.html

Fragmento:

<div id="root"></div>

Ese div es donde React carga toda la aplicacion.

React empieza desde este archivo:

frontend/src/main.jsx

Fragmento:

ReactDOM.createRoot(document.getElementById("root")).render(
  <React.StrictMode>
    <AppRouter />
  </React.StrictMode>
);

Esto significa que React toma el elemento root y dentro coloca todo el proyecto.

La pagina de inicio se modifica aqui:

frontend/src/pages/HomePage.jsx

Fragmento:

export default function HomePage() {
  return (
    <main>
      <section className="home-hero">
        <h1>Turnix</h1>
        <p>
          Gestiona filas, citas, clientes y reportes desde una base preparada para crecer.
        </p>
      </section>
    </main>
  );
}

Si queremos cambiar el texto principal de la pagina publica, lo hacemos en HomePage.jsx.


RUTAS DEL PROYECTO
------------------

Las rutas se controlan con React Router.

El archivo esta aqui:

frontend/src/router/AppRouter.jsx

Fragmento:

<Route path="/" element={<HomePage />} />
<Route path="/login" element={<LoginPage />} />
<Route path="/registro" element={<RegisterPage />} />
<Route path="/dashboard" element={<DashboardPage />} />
<Route path="/negocio" element={<BusinessPage />} />

Esto quiere decir:

/           muestra la pagina de inicio
/login      muestra el login
/registro   muestra el registro
/dashboard  muestra el dashboard
/negocio    muestra la gestion de clientes, citas, turnos y reportes

Si queremos agregar una nueva pantalla, primero creamos un archivo en:

frontend/src/pages/

Y despues lo enlazamos en AppRouter.jsx.

REGISTRO DE USUARIO
-------------------

El formulario de registro esta aqui:

frontend/src/pages/RegisterPage.jsx

Fragmento:

const [form, setForm] = useState({
  fullName: "",
  email: "",
  password: "",
  confirmPassword: "",
  role: ""
});

Aqui usamos useState, que sirve para guardar temporalmente lo que el usuario escribe en el formulario.

Cuando el usuario escribe, se ejecuta:

function handleChange(event) {
  setForm({ ...form, [event.target.name]: event.target.value });
}

Cuando el usuario presiona Registrarse, se ejecuta:

async function handleSubmit(event) {
  event.preventDefault();

  await registerUser({
    fullName: form.fullName,
    email: form.email,
    password: form.password,
    role: form.role
  });
}

Esa funcion llama a registerUser, que esta aqui:

frontend/src/api/authService.js

Fragmento:

export function registerUser(payload) {
  return apiClient.post("/auth/register", payload);
}

Eso envia los datos al backend, a esta ruta:

POST http://localhost:4000/api/auth/register

En el backend, la ruta esta aqui:

backend/src/routes/auth.routes.js

Fragmento:

router.post("/register", register);

Esa ruta llama al controlador:

backend/src/controllers/auth.controller.js

Fragmento:

const passwordHash = await bcrypt.hash(password, 10);

await pool.query(
  "INSERT INTO users (full_name, email, password_hash, role) VALUES (?, ?, ?, ?)",
  [fullName.trim(), normalizedEmail, passwordHash, role]
);

Entonces, cuando agregamos un usuario, se guarda en MySQL en la tabla:

users

La tabla esta definida aqui:

backend/database/schema.sql

Fragmento:

CREATE TABLE IF NOT EXISTS users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  full_name VARCHAR(120) NOT NULL,
  email VARCHAR(120) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  role ENUM('cliente', 'dueno_negocio', 'administrador') NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


LOGIN
-----

El login se encuentra aqui:

frontend/src/pages/LoginPage.jsx

Fragmento:

const [form, setForm] = useState({
  email: "",
  password: ""
});

Cuando el usuario inicia sesion, se ejecuta:

const response = await loginUser(form);
localStorage.setItem("turnix_user", JSON.stringify(response.data.user));
navigate("/dashboard");

La funcion loginUser esta en:

frontend/src/api/authService.js

Fragmento:

export function loginUser(payload) {
  return apiClient.post("/auth/login", payload);
}

Eso llama al backend:

POST /api/auth/login

El backend busca el usuario:

backend/src/controllers/auth.controller.js

Fragmento:

const [users] = await pool.query(
  "SELECT id, full_name, email, password_hash, role FROM users WHERE email = ? LIMIT 1",
  [email.trim().toLowerCase()]
);

Luego compara la contrasena:

bcrypt.compare(password, users[0].password_hash)

Si la contrasena es correcta, devuelve el usuario al frontend.


CONEXION CON MYSQL
------------------

La conexion a MySQL esta aqui:

backend/src/config/database.js

Fragmento:

export const pool = mysql.createPool({
  host: env.db.host,
  port: env.db.port,
  user: env.db.user,
  password: env.db.password,
  database: env.db.database
});

Los datos vienen del archivo .env:

backend/.env

Ejemplo:

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=tu_clave
DB_NAME=turnix_db

Si necesitamos cambiar el usuario o contrasena de MySQL, lo hacemos en backend/.env.


AXIOS
-----

React no se conecta directamente con MySQL. React llama al backend usando Axios.

Axios esta configurado aqui:

frontend/src/api/apiClient.js

Fragmento:

export const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_URL || "http://localhost:4000/api",
});

Esto significa que cuando escribimos:

apiClient.get("/clients")

en realidad se llama a:

http://localhost:4000/api/clients


CLIENTES
--------

La pestana de clientes esta dentro de:

frontend/src/pages/BusinessPage.jsx

Ahi se guarda la lista de clientes:

const [clients, setClients] = useState([]);

Cuando se cargan los datos, se ejecuta:

const clientsResponse = await getClients();
setClients(clientsResponse.data.clients);

La funcion getClients esta en:

frontend/src/api/clientService.js

Fragmento:

export function getClients() {
  return apiClient.get("/clients");
}

Para crear un cliente se usa:

export function createClient(payload) {
  return apiClient.post("/clients", payload);
}

En backend, las rutas estan aqui:

backend/src/routes/client.routes.js

Fragmento:

router.get("/", getClients);
router.post("/", createClient);
router.put("/:id", updateClient);
router.delete("/:id", deleteClient);

Y la logica esta aqui:

backend/src/controllers/client.controller.js

Para guardar un cliente:

await pool.query(
  "INSERT INTO clients (full_name, email, phone, document_number, notes) VALUES (?, ?, ?, ?, ?)",
  [fullName.trim(), email || null, phone || null, documentNumber || null, notes || null]
);

Los clientes se almacenan en la tabla:

clients

Definida en:

backend/database/schema.sql

Fragmento:

CREATE TABLE IF NOT EXISTS clients (
  id INT AUTO_INCREMENT PRIMARY KEY,
  full_name VARCHAR(120) NOT NULL,
  email VARCHAR(120),
  phone VARCHAR(30),
  document_number VARCHAR(40),
  notes TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

appointments

export function getReportSummary() {
  return apiClient.get("/reports/summary");
}

Backend:

backend/src/routes/report.routes.js

Controlador:

backend/src/controllers/report.controller.js

const [[clientCount]] = await pool.query("SELECT COUNT(*) AS total FROM clients");
const [[appointmentCount]] = await pool.query("SELECT COUNT(*) AS total FROM appointments");

Eso significa que el reporte cuenta los registros de la base de datos y los muestra en frontend.


DASHBOARD PRINCIPAL
-------------------

El dashboard se modifica aqui:

useEffect(() => {

    .then((response) => {
      setSummary(response.data.summary);
      setRecentAppointments(response.data.recentAppointments);
    });

useEffect sirve para ejecutar algo cuando la pagina carga.

Aqui se muestran las tarjetas:

const cards = [
  { label: "Clientes", value: summary.clients },
  { label: "Citas", value: summary.appointments },
  { label: "Citas pendientes", value: summary.pendingAppointments },

Si agregamos una cita o cliente desde la gestion, el dashboard refleja esos datos porque los lee desde MySQL mediante /api/reports/summary.


COMO SE COMUNICAN LAS PARTES
----------------------------

El flujo general es:

Usuario usa la interfaz
        ↓
React captura los datos
        ↓
Servicio Axios envia datos
        ↓
Express recibe la peticion
        ↓
Controller valida y ejecuta SQL
        ↓
MySQL guarda o devuelve datos
        ↓
Express responde
        ↓
React actualiza la pantalla

Ejemplo con clientes:

BusinessPage.jsx
        ↓
createClient()
        ↓
clientService.js
        ↓
POST /api/clients
        ↓
client.routes.js
        ↓
client.controller.js
        ↓
INSERT INTO clients
        ↓
MySQL
